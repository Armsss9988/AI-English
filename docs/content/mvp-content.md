# MVP Content Seed

## Overview

This document contains the seed content for MVP testing. Content is designed for **Vietnamese IT professionals** who need to communicate with English-speaking clients.

## Content Strategy

Target audiences:
- Developers, QA engineers, technical leads
- Working in Vietnamese companies with international clients
- Need to discuss: daily updates, bug reports, project status, technical clarifications

## Phrase Categories

1. **Greetings & Closings** — Professional email/meeting opener and closers
2. **Daily Standup** — Reporting progress, blockers, plans
3. **Bug Reports** — Describing issues clearly to clients
4. **Clarification** — Asking for more details without ambiguity
5. **ETA & Delivery** — Managing expectations on timelines
6. **Technical Discussion** — Architecture, code reviews, decisions

## Scenario Groups

### 1. Standup (5 scenarios)
- Regular daily standup
- Async standup update via chat
- Blocked standup update
- End-of-sprint summary
- Vacation/absence notice

### 2. Issue (5 scenarios)
- Bug report to client
- Production incident
- Third-party dependency issue
- Performance degradation
- Security vulnerability report

### 3. Clarification (5 scenarios)
- Unclear requirement
- Ambiguous acceptance criteria
- Conflicting instructions
- Scope clarification
- Technical approach question

### 4. ETA (5 scenarios)
- Initial timeline estimate
- Delay notification
- Rush request response
- Milestone adjustment
- Delivery confirmation

### 5. Summary (5 scenarios)
- End-of-day summary
- Weekly status report
- Sprint retrospective summary
- Project completion summary
- Handover documentation

## Sample Phrases (by category)

### Greetings
- "Good morning/afternoon, thank you for joining our call."
- "I hope this email finds you well."
- "Please let me know if you need any clarification."

### Standup
- "Yesterday I completed the authentication module."
- "I'm currently working on the API integration."
- "I need help with the database migration."
- "My plan for today is to finish testing."

### Issue Reporting
- "We've identified a critical bug in the payment flow."
- "The third-party API is returning unexpected responses."
- "There's a memory leak in the background worker."

### ETA
- "We expect to complete this by end of week."
- "The fix should take approximately 2 hours."
- "I need 3 more days to finish the feature."
